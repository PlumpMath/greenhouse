greenhouse.utils – Greenlet-Aware Concurrency Primitives¶

class greenhouse.utils.Event¶

an event for which greenlets can wait

mirrors the standard library threading.Event API

clear()¶

clear the event from being triggered

after calling this method, waiting on this event will block until the set() method has been called

isSet()¶

returns True if waiting on this event will block, False if not

is_set()¶

returns True if waiting on this event will block, False if not

set()¶

set the event to triggered

after calling this method, all greenlets waiting on this event will be woken up, and calling wait() will not block until the clear() method has been called

wait(timeout=None)¶

pause the current coroutine until this event is set

if the set() method has been called, this method will not block at all. otherwise it will block until the set() method is called.

returns True if a timeout was provided and was hit, otherwise False

class greenhouse.utils.Lock¶

an object that can only be ‘owned’ by one greenlet at a time

mirrors the standard library threading.Lock API

acquire(blocking=True)¶

lock the lock, or block until it is available

locked()¶

returns true if the lock is already ‘locked’ or ‘owned’

release()¶

open the lock back up to wake up greenlets waiting on this lock

class greenhouse.utils.RLock¶

a lock which may be acquired more than once by the same greenlet

mirrors the standard library threading.RLock API

acquire(blocking=True)¶

if the lock is owned by a different greenlet, block until it is fully released. then increment the acquired count by one

release()¶

decrement the owned count by one. if it reaches zero, fully release the lock, waking up a waiting greenlet

class greenhouse.utils.Condition(lock=None)¶

a synchronization object capable of waking all or one of its waiters

mirrors the standard library threading.Condition API

notify(num=1)¶

wake up a set number (default 1) of the waiting greenlets

you must have acquired the underlying lock first

notifyAll()¶

wake up all the greenlets waiting on the condition

you must have acquired the underlying lock first

notify_all()¶

wake up all the greenlets waiting on the condition

you must have acquired the underlying lock first

wait(timeout=None)¶

wait to be woken up by the condition

you must have acquired the underlying lock first

class greenhouse.utils.Queue(maxsize=0)¶

a producer-consumer queue

mirrors the standard library Queue.Queue API

exception Empty¶
exception Queue.Full¶
Queue.empty()¶

without blocking, returns True if the queue is empty

Queue.full()¶

returns True if the queue is full without blocking

if the queue has no maxsize, this will always return False

Queue.get(blocking=True, timeout=None)¶

get an item out of the queue

if blocking is True (default), the method will block until an item is available, or until timeout seconds, whichever comes first. if it times out, it will raise a Queue.Empty exception

if blocking is False, it will immediately either return an item or raise a Queue.Empty exception

Queue.get_nowait()¶

immediately return an item from the queue or raise Queue.Empty

Queue.join()¶

block until every put() call has had a corresponding task_done() call

Queue.put(item, blocking=True, timeout=None)¶

put an item into the queue

if blocking is True (default) and the queue has a maxsize, the method will block until a spot in the queue has been made available, or timeout seconds has passed, whichever comes first. if it times out, it will raise a Queue.Full exception

if blocking is False, it will immediately either place the item in the queue or raise a Query.Full exception

Queue.put_nowait(item)¶

immediately place an item into the queue or raise Query.Full

Queue.qsize()¶

return the number of items in the queue, without blocking

Queue.task_done()¶

mark thata job (corresponding to a put() call) is finished
